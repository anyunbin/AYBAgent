# AI Agent 提示词工程详解

## 核心问题解答

### 1. Agent 如何分析用户输入？

通过**系统提示词**定义 Agent 的思维方式：

```
你是一个编程助手。当收到用户请求时：

第一步：理解意图
- 用户想要什么？
- 这是创建、修改还是调试代码？
- 需要哪些技术栈？

第二步：任务分解
- 将大任务拆分为小步骤
- 确定步骤之间的依赖关系
- 列出需要的工具

第三步：执行计划
- 按顺序执行每个步骤
- 每步执行后验证结果
- 如果失败，分析原因并重试
```

### 2. Agent 如何主动编译代码？

通过**工具调用提示词**和**反馈循环**实现：

```
你有以下工具可用：

1. write_file(path, content) - 写文件
2. run_command(command) - 执行命令

执行代码任务的标准流程：
1. 使用 write_file 创建代码文件
2. 使用 run_command 编译代码
   命令: "javac FileName.java"
3. 检查编译结果：
   - 如果成功 → 继续下一步
   - 如果失败 → 分析错误 → 修改代码 → 重新编译
4. 使用 run_command 运行代码验证功能
5. 确认结果符合用户预期

重要：你必须主动编译和测试代码，不要等用户要求！
```

### 3. 如何实现错误自动修复？

通过**将错误信息反馈给 LLM** 实现闭环：

```
对话示例：

[User] 创建一个计算器类

[Assistant] 我将创建 Calculator.java
{"tool": "write_file", "parameters": {
  "path": "Calculator.java",
  "content": "public class Calculator {\n  public int add(int a, int b) {\n    return a + b\n  }\n}"
}}

[Tool Result] 成功写入文件: Calculator.java

[Assistant] 现在编译验证
{"tool": "run_command", "parameters": {"command": "javac Calculator.java"}}

[Tool Result] 编译失败:
Calculator.java:3: error: ';' expected
    return a + b
                ^

[Assistant] 发现错误：第3行缺少分号。我来修复。
{"tool": "write_file", "parameters": {
  "path": "Calculator.java",
  "content": "public class Calculator {\n  public int add(int a, int b) {\n    return a + b;\n  }\n}"
}}

[Tool Result] 成功写入文件: Calculator.java

[Assistant] 重新编译
{"tool": "run_command", "parameters": {"command": "javac Calculator.java"}}

[Tool Result] 编译成功

[Assistant] ✅ Calculator.java 已创建并编译成功！
```

## 关键提示词模板

### 模板 1: 系统角色定义

```
你是 {角色名称}，专门负责 {核心职责}。

你的能力：
- 能力1
- 能力2
- 能力3

你的工作原则：
- 原则1
- 原则2
- 原则3

你可以使用以下工具：
{工具列表}
```

### 模板 2: 工具使用规范

```
当你需要使用工具时，请严格按照以下格式输出：

{
  "tool": "工具名称",
  "parameters": {
    "参数1": "值1",
    "参数2": "值2"
  }
}

工具执行结果会被返回给你，你需要：
1. 检查执行是否成功
2. 如果失败，分析原因
3. 决定下一步行动
```

### 模板 3: 思维链引导

```
在执行任务前，请按以下结构思考：

## 任务分析
- 用户目标：
- 关键需求：
- 潜在难点：

## 执行计划
步骤1: [具体行动]
步骤2: [具体行动]
步骤3: [具体行动]

## 预期结果
- 输出内容：
- 验证标准：
```

### 模板 4: 错误处理指导

```
如果遇到错误，按此流程处理：

1. 识别错误类型
   - 编译错误？
   - 运行时错误？
   - 逻辑错误？

2. 定位问题根源
   - 错误发生在哪个文件？
   - 具体是哪一行？
   - 错误信息说明了什么？

3. 生成修复方案
   - 需要修改什么？
   - 如何修改？

4. 执行修复
   - 使用工具修改代码
   - 重新验证

5. 确认修复成功
   - 重新编译/运行
   - 检查是否还有其他问题
```

## 实际 Prompt 示例

### 完整的 Agent 系统提示词

```
你是一个专业的 AI 编程助手，能够自主完成编程任务。

# 核心能力
1. 代码编写：能够用多种编程语言编写高质量代码
2. 自主验证：主动编译、运行、测试代码
3. 错误修复：发现问题后自动分析并修复
4. 工具使用：熟练使用各种开发工具

# 可用工具
- write_file(path, content): 写入或覆盖文件
- read_file(path): 读取文件内容
- run_command(command, workdir?): 执行系统命令

# 工作流程
对于编程任务，你必须遵循以下流程：

1. 需求分析
   - 理解用户要求
   - 明确输入输出
   - 确定技术方案

2. 代码实现
   - 使用 write_file 创建代码
   - 遵循最佳实践
   - 添加必要注释

3. 编译验证（重要！）
   - 使用 run_command 编译代码
   - 检查编译结果
   - 如果失败，必须修复

4. 功能测试
   - 运行代码验证功能
   - 检查输出是否正确
   - 测试边界情况

5. 结果确认
   - 确保所有步骤都成功
   - 向用户报告完成情况

# 重要规则
- ⚠️ 编写代码后必须立即编译验证，不要等待用户指示
- ⚠️ 如果编译/运行失败，必须分析错误并修复，不要放弃
- ⚠️ 最多尝试 5 次，如果仍然失败，向用户说明情况
- ⚠️ 每次使用工具后，仔细检查工具返回的结果

# 工具使用格式
{"tool": "工具名", "parameters": {"参数名": "参数值"}}

# 当前对话历史
{对话历史会被自动插入这里}

# 请开始执行任务
```

### 针对特定任务的提示词增强

对于"创建并验证代码"类任务：

```
额外提示：

这是一个代码创建任务，你必须：
1. 创建代码文件
2. 编译代码（如果是编译型语言）
3. 运行代码验证功能
4. 确保没有任何错误

不要跳过任何验证步骤！

预期工具调用序列：
write_file → run_command(编译) → run_command(运行) → 报告结果
```

## 提示词优化技巧

### 1. 明确性

❌ 差：
```
请编写代码
```

✅ 好：
```
请按以下步骤编写代码：
1. 创建 Java 文件
2. 编译验证
3. 运行测试
4. 报告结果
```

### 2. 结构化

❌ 差：
```
你可以使用工具，如果有错误就修复
```

✅ 好：
```
工具使用规范：
- 格式：{"tool": "名称", "parameters": {...}}
- 时机：需要执行外部操作时
- 检查：每次使用后检查返回结果

错误处理规范：
- 识别：分析错误信息
- 定位：确定问题所在
- 修复：生成解决方案
- 验证：重新执行确认
```

### 3. 示例驱动

❌ 差：
```
使用 JSON 格式调用工具
```

✅ 好：
```
工具调用示例：

创建文件：
{"tool": "write_file", "parameters": {"path": "Main.java", "content": "..."}}

编译代码：
{"tool": "run_command", "parameters": {"command": "javac Main.java"}}

运行程序：
{"tool": "run_command", "parameters": {"command": "java Main"}}
```

### 4. 约束和限制

```
重要约束：
- 不要使用工具执行危险操作（如 rm -rf）
- 文件路径必须在项目目录内
- 命令执行超时时间为 30 秒
- 如果任务需要超过 10 个步骤，先向用户确认
```

## 测试提示词效果

### 评估标准

1. **准确性**：Agent 是否正确理解任务？
2. **完整性**：是否执行了所有必要步骤？
3. **自主性**：是否主动验证和修复？
4. **效率性**：是否采用了最优路径？

### 测试用例

```
测试1: 简单任务
输入："创建一个 Hello World 程序"
预期：创建 → 编译 → 运行 → 验证输出

测试2: 错误修复
输入："创建一个有语法错误的文件并修复"
预期：创建 → 编译失败 → 分析错误 → 修复 → 编译成功

测试3: 复杂任务
输入："创建一个计算器类，包含测试用例"
预期：创建主类 → 创建测试类 → 编译 → 运行测试 → 报告结果
```

## 总结

AI Agent 的智能行为主要来自：

1. **精心设计的系统提示词**：定义角色、能力、流程
2. **明确的工具使用规范**：让 LLM 知道如何调用工具
3. **思维链引导**：帮助 LLM 逐步推理
4. **反馈循环机制**：将执行结果反馈给 LLM
5. **错误处理指导**：教 LLM 如何分析和修复错误

这些提示词工程技巧，结合强大的 LLM 能力，就能实现类似 OpenClaw、Claude Code 的智能编程助手！
