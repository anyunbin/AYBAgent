# AI Agent 快速开始指南

## 🎯 核心问题解答

### 1. Agent 如何工作？

**3 个关键机制**：

1. **Prompt Engineering（提示词工程）**
   - 告诉 LLM 它是谁、能做什么、该怎么做
   - 例：「你是编程助手，编写代码后必须编译验证」

2. **Tool Calling（工具调用）**
   - LLM 通过调用工具与外界交互
   - 例：`write_file()`、`run_command()`

3. **Feedback Loop（反馈循环）**
   - 工具执行结果反馈给 LLM
   - LLM 根据结果决定下一步
   - 形成闭环，实现自主纠错

### 2. 如何主动编译代码直到成功？

**关键：在系统提示词中明确要求**

```
系统提示词中写明：
"编写代码后，你必须：
1. 使用 run_command 编译
2. 检查编译结果
3. 如果失败，分析错误并修复
4. 重复直到成功"
```

**执行流程**：

```
Step 1: LLM 写代码 → write_file("Main.java")
Step 2: LLM 编译 → run_command("javac Main.java")
Step 3: 返回"编译失败: line 10 缺少分号"
Step 4: LLM 看到错误，决定修复
Step 5: LLM 修改代码 → write_file("Main.java", 修复后)
Step 6: LLM 重新编译 → run_command("javac Main.java")
Step 7: 返回"编译成功" → 任务完成
```

### 3. 涉及大量提示词工程吗？

**是的！4 种核心提示词**：

| 提示词类型 | 作用 | 示例 |
|-----------|------|------|
| 系统提示词 | 定义角色和原则 | 「你是编程助手，必须编译验证代码」 |
| 工具定义 | 说明可用工具 | 「write_file(path, content) - 写文件」 |
| 对话历史 | 提供上下文 | 「[user]: 创建类」「[tool]: 编译失败」 |
| 思维链 | 引导推理 | 「思考: 1)当前状态 2)下一步行动」 |

## 🚀 5 分钟理解 Agent

### 核心代码（简化版）

```java
// 1. Agent 主循环
while (未完成) {
    // 构建提示词
    String prompt =
        系统提示词 +
        工具定义 +
        对话历史 +
        思维链引导;

    // 调用 LLM
    Response response = llm.chat(prompt);

    // 如果需要使用工具
    if (response.是工具调用) {
        // 执行工具
        String result = 执行工具(response.工具名, response.参数);

        // 保存结果到历史
        对话历史.add(result);

        // 继续循环，LLM 会根据结果决定下一步
    } else {
        // LLM 认为任务完成
        return response.内容;
    }
}
```

## 🎨 关键提示词示例

### 系统提示词模板

```
你是一个专业的 AI 编程助手，能够自主完成编程任务。

核心能力：
- 代码编写
- 自主验证（编译、运行、测试）
- 错误修复

工作原则：
- 编写代码后必须立即验证
- 如果编译/运行失败，必须修复
- 最多尝试 5 次
- 不要等待用户指示

可用工具：
- write_file(path, content): 写入文件
- read_file(path): 读取文件
- run_command(command): 执行命令

工具使用格式：
{"tool": "工具名", "parameters": {"参数": "值"}}

执行流程：
1. 创建代码文件
2. 编译验证
3. 如果失败 → 分析错误 → 修复 → 重新验证
4. 如果成功 → 运行测试
5. 报告结果
```

### 错误处理提示词

```
当遇到错误时：

1. 识别错误类型
   - 编译错误（语法问题）
   - 运行时错误（逻辑问题）
   - 测试失败（功能问题）

2. 定位问题
   - 仔细阅读错误信息
   - 确定是哪个文件的哪一行

3. 生成解决方案
   - 分析根本原因
   - 提出修复方案

4. 执行修复
   - 使用 write_file 修改代码
   - 重新验证

不要放弃！最多尝试 5 次。
```

## 🎓 核心要点总结

### Agent 的本质

```
Agent = LLM + Tools + Prompts + Feedback Loop
```

### 智能来源

1. **70% 来自提示词**：精心设计的提示词引导 LLM 行为
2. **20% 来自 LLM**：强大的语言理解和推理能力
3. **10% 来自工具**：提供实际执行能力

### 成功的关键

1. **明确的系统提示词**：清晰定义角色和流程
2. **完善的工具系统**：提供足够的执行能力
3. **有效的反馈循环**：让 LLM 能够自我修正
4. **合理的错误处理**：应对各种异常情况
