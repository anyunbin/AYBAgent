# AI Agent 实现指南 - 从零构建你的智能编程助手

## 项目概述

本项目展示了如何实现一个完整的 AI Agent 框架，类似于 OpenClaw、Claude Code 等智能编程助手。

## 核心问题解答

### Q1: Agent 如何分析和执行用户输入？

**答案**: 通过 **Prompt Engineering（提示词工程）** + **反馈循环机制**

```
用户输入
  ↓
构建提示词（系统提示 + 工具定义 + 对话历史）
  ↓
调用 LLM 分析
  ↓
LLM 决定使用哪个工具
  ↓
执行工具（如 write_file、run_command）
  ↓
将结果反馈给 LLM
  ↓
LLM 根据结果决定下一步
  ↓
重复直到任务完成
```

### Q2: Agent 如何主动编译代码直到成功？

**答案**: 通过 **错误反馈循环** + **自我修复提示词**

```java
// 系统提示词中包含明确指示：
"""
编写代码后必须：
1. 使用 run_command 编译代码
2. 检查编译结果
3. 如果失败，分析错误信息
4. 修复代码
5. 重新编译
6. 重复直到成功
"""
```

**实际执行流程**：

```
步骤1: LLM 创建代码文件
  write_file("Calculator.java", "代码内容")

步骤2: LLM 主动编译
  run_command("javac Calculator.java")
  结果: "编译失败: line 10: ';' expected"

步骤3: LLM 收到错误信息
  对话历史中包含: "[tool] 编译失败: ..."

步骤4: LLM 分析错误并修复
  识别: 第10行缺少分号
  决定: 修改代码添加分号

步骤5: LLM 重新写入修复后的代码
  write_file("Calculator.java", "修复后的代码")

步骤6: LLM 再次编译
  run_command("javac Calculator.java")
  结果: "编译成功"

步骤7: 任务完成
```

### Q3: 涉及大量提示词工程吗？

**答案**: 是的！提示词工程是 Agent 的核心灵魂！

主要包含 4 类提示词：

#### 1. 系统提示词（System Prompt）
定义 Agent 的角色和行为规范

```
你是一个专业的编程助手 Agent。

你的能力：
- 编写代码
- 编译验证
- 自动修复错误
- 运行测试

工作原则：
- 编写代码后必须立即编译验证
- 如果编译失败，必须分析错误并修复
- 最多尝试 5 次
- 不要等待用户指示，主动完成所有步骤
```

#### 2. 工具定义提示词（Tool Definition）
告诉 LLM 有哪些工具可用

```
可用工具：

工具名: write_file
描述: 写入文件内容
参数: path(文件路径), content(文件内容)

工具名: run_command
描述: 执行系统命令（编译、运行等）
参数: command(要执行的命令)

使用格式:
{"tool": "工具名", "parameters": {"参数": "值"}}
```

#### 3. 思维链提示词（Chain of Thought）
引导 LLM 逐步推理

```
请按以下步骤思考：
1. 当前任务状态是什么？
2. 下一步应该做什么？
3. 需要使用哪个工具？
4. 预期结果是什么？
5. 如果上一步有错误，如何修复？
```

#### 4. 上下文提示词（Context）
提供对话历史

```
对话历史:
[user]: 创建一个计算器类
[assistant]: 我将创建 Calculator.java
[tool]: 文件创建成功
[assistant]: 现在编译验证
[tool]: 编译失败 - 第10行缺少分号
[assistant]: 我发现错误，正在修复...
```

## 核心代码解析

### 1. Agent 主循环

```java
public String execute(String userInput) {
    memory.addUserMessage(userInput);

    for (int i = 0; i < maxIterations; i++) {
        // 1. 构建完整提示词
        String prompt = buildPrompt();

        // 2. 调用 LLM
        LLMResponse response = llmClient.chat(prompt);

        // 3. 判断响应类型
        if (response.isToolCall()) {
            // 需要调用工具
            String result = executeToolCall(response.getToolCall());
            memory.addToolResult(result);

            // 检查是否完成
            if (isTaskComplete(result)) {
                return "任务完成";
            }
            // 否则继续循环，LLM 会根据工具结果做下一步决策
        } else {
            // LLM 认为任务完成
            return response.getContent();
        }
    }
}
```

### 2. 提示词构建

```java
private String buildPrompt() {
    StringBuilder prompt = new StringBuilder();

    // 第1层: 系统提示词
    prompt.append("""
        你是一个专业的编程助手。
        你的任务是帮助用户完成编程任务。

        你必须遵循以下流程：
        1. 编写代码
        2. 编译验证
        3. 如果失败，修复错误
        4. 重复直到成功
        """);

    // 第2层: 工具定义
    prompt.append("可用工具：\n");
    for (Tool tool : toolRegistry.getAllTools()) {
        prompt.append(String.format(
            "- %s: %s\n",
            tool.getName(),
            tool.getDescription()
        ));
    }

    // 第3层: 对话历史
    prompt.append(memory.getConversationHistory());

    // 第4层: 思维链引导
    prompt.append("""
        请思考：
        1. 当前任务状态？
        2. 下一步行动？
        3. 需要哪个工具？
        """);

    return prompt.toString();
}
```

### 3. 工具执行

```java
private String executeToolCall(ToolCall toolCall) {
    Tool tool = toolRegistry.getTool(toolCall.getToolName());

    try {
        // 执行工具
        String result = tool.execute(toolCall.getParameters());
        return result;
    } catch (Exception e) {
        // 错误也要返回给 LLM，让它知道出了什么问题
        return "工具执行失败: " + e.getMessage();
    }
}
```

## 关键机制详解

### 1. 反馈循环（Feedback Loop）

这是 Agent 自主性的关键！

```
LLM 决策 → 工具执行 → 结果反馈 → LLM 再决策
   ↑                                    ↓
   └────────────────────────────────────┘
```

**示例**：

```
迭代1:
  LLM: 创建文件
  工具: write_file → 成功
  反馈: "文件创建成功"

迭代2:
  LLM: 编译代码（因为看到文件创建成功）
  工具: run_command → 失败（缺少分号）
  反馈: "编译失败: ';' expected"

迭代3:
  LLM: 修复代码（因为看到编译错误）
  工具: write_file → 成功
  反馈: "文件修改成功"

迭代4:
  LLM: 重新编译（因为代码已修复）
  工具: run_command → 成功
  反馈: "编译成功"

迭代5:
  LLM: 任务完成（因为看到编译成功）
  返回: "✅ 所有步骤已完成"
```

### 2. 自我修复（Self-Correction）

通过错误信息反馈实现：

```java
// 编译失败的错误信息会被添加到对话历史
memory.addToolResult("run_command", "编译失败: line 10: ';' expected");

// 下次调用 LLM 时，提示词中会包含这个错误信息
// LLM 看到错误后会：
// 1. 识别错误类型（语法错误）
// 2. 定位问题位置（第10行）
// 3. 生成修复方案（添加分号）
// 4. 使用 write_file 工具修复代码
```

### 3. 任务状态追踪

```java
private boolean isTaskComplete(String toolResult) {
    // 简单版本：检查是否有错误
    return !toolResult.toLowerCase().contains("error")
        && !toolResult.toLowerCase().contains("failed");
}

// 进阶版本可以：
// - 检查特定的成功标志
// - 验证输出格式
// - 运行测试用例
```

## 最佳实践

### 1. 提示词设计

✅ **好的提示词**：
- 明确具体的指令
- 包含示例
- 定义清晰的流程
- 设置合理的约束

❌ **差的提示词**：
- 模糊不清
- 缺少示例
- 没有流程指导
- 约束条件不足

### 2. 错误处理

```java
// 提示词中明确错误处理流程
"""
如果遇到错误：
1. 仔细阅读错误信息
2. 识别错误类型（编译错误、运行时错误、逻辑错误）
3. 定位问题根源
4. 生成修复方案
5. 执行修复
6. 重新验证
"""
```

### 3. 安全控制

```java
// 工具执行前检查
```

## 常见问题

### Q: LLM 不按提示词执行怎么办？

A:
1. 增强提示词的明确性
2. 提供更多示例
3. 使用更强大的模型（如 GPT-4）
4. 通过 few-shot 示例引导

### Q: 如何控制 Agent 的行为？

A:
1. 通过系统提示词定义规则
2. 限制可用工具
3. 设置最大迭代次数
4. 添加中间审核步骤

### Q: 如何提高 Agent 的成功率？

A:
1. 优化提示词设计
2. 增加错误处理逻辑
3. 提供详细的工具说明
4. 使用更好的 LLM 模型

## 总结

实现 AI Agent 的核心要素：

1. **Prompt Engineering**: 精心设计的提示词是灵魂
2. **Tool System**: 工具系统是手和脚
3. **Feedback Loop**: 反馈循环是神经系统
4. **Memory Management**: 记忆管理是大脑
5. **LLM**: 大语言模型是智能核心

通过这些机制的有机结合，Agent 就能像人类程序员一样：
- 理解需求
- 编写代码
- 自主验证
- 发现错误
- 修复问题
- 直到成功

这就是 OpenClaw、Claude Code、Cursor 等 AI 编程助手背后的核心原理！
